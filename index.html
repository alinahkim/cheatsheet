CSS Selectors Cheatsheet

>
Selector Syntax Description Example Example Description Specificity
Type Selector (element) {
property: value;
}		 Selects an element in HTML p {
color: green;
}		 The element type is <p> (paragraph). The <p> elements would become green (foreground). Lowest specificity.
Universal Selector * {
property: value;
}		 Selects elements of any type/all elements on a page. Or, it can select all children of a parent element. * {
font-family: Verdana;
}		 Every text element on the page will have its font changed to Verdana. No specificity.
Class Attribute Selector .(class) {
property: value;
}		 Targets elements based on its attribute in HTML. In HTML, the attribute will be class='value' and the value is specified after the period(.) in this CSS selector. An HTML element's class can have multiple values and thus we can mix and match CSS classes. .brand {
font-weight: bold;
}		 Elements with a class of 'brand' will be bolded. More specific than Type Selectors.
ID Selector #(ID) {
property: value;
}		 Targets single elements with specific ID attribute value. If an HTML element needs to be styled uniquely, use this. An element's ID can only have a single value and can only be used once per page, so an ID only styles one element. Use a dash between multiple words in an ID: id=’large=title’ #large-title {
text-transform: uppercase;
}		 The element with the ID of 'large-title' will be in all uppercase letters. More specific than type and class selectors. Overrides the style of these selectors. ID selectors are more specific than any selector using only element type and/or class selectors -- this includes chaining and descendant combinator selectors.
Attribute Selector [attribute] {
property: value;
}
or
type[attribute*='value'] {
property: value;
}		 (1)Targets HTML elements that already contain attributes. Selects the attribute instead of its value. (2)Targets elements with an attribute and value by adding type and attribute values. This selects an element that has an attribute with any instance of a specified word in its value. Both alleviate the need to add new code/HTML markup. [href] {
color: magenta;
}
or
img[src*='winter'] {
height: 50px;
}		 (1)All elements with an href attribute will have magenta text. (2) <img src='/images/seasons/cold/winter.jpg'> is what is in the HTML. The example looks for an <img> element with an attribute of src that contains the string 'winter.' (1) is more specific than type selectors. (2) is very specific.
Pseudo-class Selector (element):pseudo-class {
property: value;
}		 Used to change the appearance of certain elements after user interactions. Some pseudo-classes: :focus, :visited, :disabled, and :active. A pseudo-class can be attached to any selector. p:hover {
background-color: lime;
}		 Whenever the mouse hovers over a paragraph element, that paragraph will have a lime-colored background. n/a
Chaining Selector (element).class {
property: value;
}		 Targets a specific HTML element that has two or more CSS selectors at the same time by combining multiple selectors. h1.special {
font-family: 'Times New Roman';
}		 <h1> elements with a class of 'special' will be in Times New Roman font. More specific than type and class selectors.
Descendant Combinator Selector (selector1: element/class/attribute) (selector2: element/class/attribute) {
property: value;
}		 Selects elements nested within other HTML elements (AKA descendants). Can be nested elements, does not need a class. .main-list li {
font-style: italic;
}		 In the HTML, li elements are descendents of a ul element with a class of 'main-list'. This selects for all list elements nested within elements with a 'main-list' class. More specific than type and class selectors.
